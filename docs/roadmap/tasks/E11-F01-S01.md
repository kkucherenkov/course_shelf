## Notes

- Completed: 2026-04-27
- Result: T-2026-04-27-033 — `feat/web-tokens-page` — commits `ff0f21c` (dark default) + `4f09056` (tokens canvas + vitest infra). 6 tests; lint + typecheck clean.
- Tailwind 4 wiring uses `@tailwindcss/vite` directly (vite plugin) instead of the `@nuxtjs/tailwindcss` Nuxt module the card mentioned. Reason: the Nuxt module had not yet shipped a Tailwind 4-compatible release at scaffold time; the official path with Tailwind 4 is the vite plugin. Same `@theme` capability, fewer moving parts.
- Dark theme is the default via `colorMode: { preference: 'dark', fallback: 'dark' }` in `nuxt.config.ts` (`@nuxtjs/color-mode` ships transitively with Nuxt UI).
- `pages/__tokens.vue` is a developer-only foundations canvas (no i18n, plain English labels). Five categories: color (brand/surface/border/text/status with live `getComputedStyle` values that re-read on theme toggle), spacing bars, radius chips, typography specimens per `--role-*`, motion + opacity raw `<dl>`. `UColorModeButton` in the header for one-click theme verification.
- Vitest infra adjustment: added `vitest.nuxt-imports-shim.ts` + `@vitejs/plugin-vue` dev-dep so `#imports` resolves in the plain vitest environment without spinning up the full Nuxt runtime. Pages are component-tested via `@vue/test-utils` `mount` with `UColorModeButton` stubbed; integration with `mountSuspended` deferred to a richer test infra slice later.
