## Goal

Every HTTP call goes through `@app/api-client-ts`; bearer token resolved from a Pinia auth store backed by Better Auth client.

## Acceptance

- `stores/auth.ts` Pinia store wraps `better-auth/client`; exposes `signIn`, `signOut`, `session`
- `createClient` from `@app/api-client-ts` is configured with a bearer interceptor that reads from the auth store
- All requests to `/api/v1/*` include `Authorization: Bearer <token>`
- On `401`, the interceptor attempts token refresh via Better Auth; if that fails, clears session and routes to `/sign-in`
- No `useApi`, `useAuth`, or `useAuthToken` composables
