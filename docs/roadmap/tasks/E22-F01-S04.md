Reconciled 2026-07-17 (issue #164). Status stays ⬜ Not started because this
card's actual deliverables — **a PR comment with diff thumbnails** and **a
required `visual-approved` label gate** — do not exist. What _does_ exist (and
what #164 mistook for this card shipping) is the surrounding baseline estate,
landed by other work:

- `.github/workflows/quality.yml` runs the Storybook test-runner visual
  regression on every PR (added when the PR gates were fixed, PR #154). It
  **fails the check** when a story's pixels drift — but it does not post diff
  thumbnails, and there is no label to override it.
- `.github/workflows/regen-snapshots.yml` captures/commits jammy baselines on
  demand.
- Committed baselines live under `packages/ui/test/__snapshots__/` and
  `tests/e2e/*-snapshots/`.

So the visual-diff _detection_ is live; this card's remaining scope is the
**human-in-the-loop approval UX** on top of it: thumbnail comments + a
`visual-approved` label that unblocks merge when a diff is intentional. Pick it
up as that, not as greenfield.
