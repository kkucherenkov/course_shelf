## Goal

Mobile mirrors the web bearer flow. **Email + password is the PRIMARY sign-in
path**; phone-OTP (Better Auth `phone-number/send-otp` + `verify-otp`) is the
SECONDARY path, reached from a "sign in with phone instead" link. This matches
the 2026-07-15 mobile-foundations closeout decision (which reversed the original
card) and the "web is the tiebreaker" rule вАФ `apps/web` is email/password only.

## Acceptance

- `AuthCubit` emits `unauthenticated`, `authenticating`, `authenticated`, `otpSent`, `error` (note: `error`, not `Failed`; `otpSent` was omitted from the original card)
- Token persists across restarts
