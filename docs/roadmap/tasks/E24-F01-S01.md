## Goal

Realtime spine: clients receive a short-lived Centrifugo JWT (TTL ≤5 min) from `POST /api/v1/realtime/token`; AsyncAPI 3.0 documents the channels they subscribe to.

## Acceptance

- `packages/specs/asyncapi/centrifugo.yaml` describes core channels (e.g. `library:scan`, `notes:lesson:{id}`)
- `POST /api/v1/realtime/token` declared in OpenAPI; returns `{ token, expiresAt }`
- Token TTL ≤5 minutes; signed with `CENTRIFUGO_TOKEN_HMAC` from `AppConfig`
- Endpoint requires authenticated session; per-user `sub` claim in the JWT payload

## Notes

- Completed: 2026-04-27
- Result: T-2026-04-27-029 — three commits on `feat/realtime-channels`: `f1e30d2` (asyncapi expansion), `9007b95` (controller refactor + JWT channels claim + tests), `fd2905d` (centrifugo namespace config). Backend tests 655/655; lint + typecheck clean.
- Controller uses the `@Session()` decorator (E04 alignment) — global `SessionGuard` returns 401 before the service is reached, so `RealtimeAuthError` is dead code on the controller path. Kept the class file for future callers; documented the change in `realtime.errors.ts`.
- JWT payload: `{ sub: user.id, exp: now + 300, channels: ['system:health', 'progress:user:<id>', 'notifications:user:<id>'] }`. The `notes:lesson:*` and `library:scan:*` channels are deliberately **not** in the connection token's `channels` claim — those need subscribe-time per-channel authorization (the Centrifugo `subscribe_proxy` flow), which is a future story.
- AsyncAPI deviation: the `discriminator` block in `ScanEvent` was omitted because AsyncAPI 3 / JSON-Schema-Draft-7 does not recognise it; per-variant `kind` enums on each `oneOf` branch is sufficient for client-side discrimination.
- Centrifugo namespaces declared in `docker/centrifugo/config.json`: `system` (existing, presence + history), `library`/`notes`/`progress`/`notifications` (presence off, history off — safe defaults; future stories can flip these on per channel as needed).
- Throttle: `@Throttle({ default: { limit: 30, ttl: 60_000 } })` — 30 mints/min/user is generous for flaky-network reconnects but blocks runaway loops.
- Out of scope (future stories): publishing actual events into the new channels (`CentrifugoService.publish` is wired but no module currently calls it) and subscribe-time per-channel auth for the wildcard channels.
