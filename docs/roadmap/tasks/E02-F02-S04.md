## Notes

- Completed: 2026-04-27 (retroactive flip via T-2026-04-27-030)
- Deviation 1: contract testing uses **Schemathesis** (`schemathesis/schemathesis:stable` Docker image), not Dredd or Prism. Schemathesis is property-based: it generates `--hypothesis-max-examples=25` random requests per operation against a running backend at `host.docker.internal:3000`. Stronger guarantees than Dredd's example-driven approach.
- Deviation 2: there is no `apps/backend/test/contract/` directory. The `spec:contract-test` script under `packages/specs/scripts/contract-test.ts` orchestrates the docker-run; the backend is exercised over the wire as a black box. CI runs this against a freshly-spun docker compose stack.
- "CI gate: paths.* > test files count" is implicitly satisfied — Schemathesis enumerates operations from the spec at runtime; you cannot ship a route that exists in code but not in the spec because the validator middleware would 404 it (and vice versa, you cannot ship a spec route without an implementation because Schemathesis would surface a 5xx).
