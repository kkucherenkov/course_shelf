## Goal

Runtime validation of request/response against the bundled spec; `/api/v1/auth/*` and `/healthz` exempted.

## Acceptance

- Invalid request body returns RFC 9457 problem JSON
- Schema mismatches in responses fail in dev/test, log-only in prod
- Better Auth routes under `/api/v1/auth/*` are not blocked

## Notes

- Completed: 2026-04-27
- Result: T-2026-04-27-028 — commit `10dd112` on `feat/e04-alignment`.
- `ignorePaths` regex anchored at the start of the validator's mount path: `/^\/v1\/auth(\/|$)/` (the validator mounts on `/api`, so paths inside the regex are relative to that mount). Existing `/v1/stream/lessons/` exemption (binary range responses) preserved.
- `/healthz` and `/readyz` are not listed in `ignorePaths`: they're excluded from `setGlobalPrefix('api')` so they sit at the server root and never reach the validator middleware in the first place.
- Validator errors flow through the global `HttpExceptionFilter` which produces RFC 9457 `application/problem+json` bodies with localised `title` + `detail`.
