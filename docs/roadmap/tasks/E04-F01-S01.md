## Goal

An app skeleton that compiles, with bounded-context module scaffolding and the shared kernel (Branded IDs + DomainError) ready.

## Acceptance

- `apps/backend/src/{main.ts,app.module.ts}` boot
- Empty modules: CatalogModule, AccessModule, LearningModule, StreamingModule, OpsModule
- PrismaModule + service in `infrastructure/prisma/`
- `apps/backend/prisma/schema.prisma` exists with provider toggle via env
- `apps/backend/src/shared/branded-id.ts` exports `Branded<T>` — `Id<'Course'>` and `Id<'Lesson'>` cannot be assigned to each other
- `apps/backend/src/shared/domain-error.ts` exports `DomainError` base + named subclasses (`InvariantViolation`, `NotFound`, `PermissionDenied`)
- A Nest `DomainErrorFilter` maps each subclass to its HTTP status code
- 100% test coverage on the shared kernel types
- Dev stack runs via Docker Compose (`docker compose -f docker/compose.yml up -d`); no separate `dev:api` script needed
