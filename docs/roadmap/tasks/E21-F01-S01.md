## Notes

- Completed: 2026-04-27
- Result: T-2026-04-27-032 — three commits on `feat/admin-dashboard`: `a89fc9c` (spec), `1b3972d` (codegen), `d9bbe11` (impl). Backend tests 664/664; lint + typecheck clean.
- Endpoint: `GET /api/v1/admin/dashboard` returns `AdminDashboardDto { generatedAt, counts: { libraries, users, courses, lessons }, latestScan: AdminDashboardLatestScan | null, errorsLast24h }`. Six counts run in parallel via `Promise.all`.
- Layering: instead of injecting `PrismaService` directly into the application handler (which would trip the `no-restricted-syntax` lint rule banning Prisma imports in `application/**`), the work was structured as a proper port (`apps/backend/src/modules/admin/domain/dashboard.port.ts`) + adapter (`apps/backend/src/modules/admin/infra/prisma-dashboard.adapter.ts`). Tighter DDD layering than the brief asked for.
- `errorsLast24h` uses `Scan.startedAt > now - 24h` as a proxy timestamp because `ScanErrorRecord` has no own timestamp. Documented in the spec description and the adapter code.
- Auth chain: global `SessionGuard` (E04 alignment) handles 401, then class-level `@UseGuards(AdminGuard)` handles 403. Future `/admin/*` routes inherit the AdminGuard automatically.
