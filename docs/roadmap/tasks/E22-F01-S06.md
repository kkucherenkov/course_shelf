## Goal

Tagged release builds and publishes versioned Docker images to a
public registry, generates a Conventional-Commits changelog, and
creates a Forgejo Release with a deploy-ready artefact bundle.

## Acceptance

- Images on GitHub Container Registry under
  `ghcr.io/<ns>/courseshelf-{backend,web}:{tag,major.minor,major,latest}`.
  Proxy + centrifugo run upstream images directly, no custom builds.
- amd64 only for v1; arm64 deferred to a follow-up story.
- Forgejo Release page populated with a Conventional-Commits-derived
  changelog (git-cliff) and a deploy-ready artefact bundle
  (`compose-release-v<semver>.yml`, `.env.release.example`, prod
  nginx config, release notes).

## Design decision тАФ registry choice

Implementation pushes to GHCR (`ghcr.io`) rather than the homelab
Forgejo OCI endpoint (`code.homelab.local`) called out in the
original card. Rationale (mirrored verbatim from
`.forgejo/workflows/release.yml`):

> The homelab Forgejo at code.homelab.local doesn't terminate TLS on
> its container-registry endpoint. Docker requires HTTPS by default,
> and the only workaround (insecure-registries via daemon.json +
> restart) needs systemd, which the act_runner's container init
> isn't. GHCR is reliable public infra, has proper TLS, and the
> runner already pulls FROM-images from Docker Hub fine тАФ it has
> internet access and HTTPS works.

Moving to Forgejo OCI is deferred until the homelab registry gains a
working TLS cert; it's tracked as a follow-up rather than blocking
this card.
