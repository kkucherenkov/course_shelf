## Goal

Visual home for OAuth/SSO providers — rendered when the admin has configured ≥1 provider.

## Acceptance

- Props: `providers: Array<{ id: 'google' | 'github' | 'sso'; label: string; iconName: IconName }>`
- Renders one `AppButton` (secondary) per provider with the configured icon + label
- When `providers` is empty, the component renders nothing (auth pages also hide the surrounding `or` divider)
- States in stories: Default (3 providers as in bundle), GoogleOnly, GitHubOnly, SSOOnly, Empty

## Notes

**v1 scope.** PRD lists OAuth/OIDC/SAML at v2+. v1 ships this component with an empty `providers` list by default — the visual is in place for when v2 lands. When the admin enables Better Auth's `genericOAuth` or per-provider plugins (post-v1), the auth page populates `providers` from `GET /admin/instance` and the SsoBlock renders. Until then, the divider and SsoBlock are hidden on the auth pages.
