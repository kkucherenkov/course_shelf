## Notes

- Completed: 2026-04-27
- Result: T-2026-04-26-027 — three commits on `feat/progress-batch`: `ff99972` (spec), `e47802a` (codegen), `ac95b71` (impl). Backend tests 638/638; lint + typecheck clean.
- Per-item outcome: `accepted` (write applied) | `stale` (server `lastSeenAt > clientUpdatedAt`; server state returned so the offline client can overwrite local cache) | `forbidden` (no READ grant or lesson missing — collapsed for no-oracle).
- Items processed **sequentially** (not Promise.all): bounded fan-out (≤200), keeps Prisma writes serial, avoids re-entering the same `(userId, lessonId)` row in the duplicate-item edge case.
- Equal timestamps (`lastSeenAt === clientUpdatedAt`) classify as `accepted`, not `stale` — only strictly-greater server time means the client cache is behind.
- Non-`PermissionDenied` errors bubble (the whole batch 500s); validation errors never reach the handler because the OpenAPI validator rejects malformed bodies upstream.
