- Completed: 2026-07-16
- Result: PR (pending) — branch `feat/e16-f01-s01-widgetbook`; design
  `docs/superpowers/specs/2026-07-16-e16-f01-s01-widgetbook-bootstrap-design.md`
- Decisions (see design doc): the runnable entrypoint lives in `apps/mobile`
  (the only Flutter app with platform runners), not `packages/ui_flutter`; the
  catalog still documents `app_ui` by importing it. `always_use_package_imports`
  forces catalog logic under `lib/widgetbook/` with a thin `widgetbook/main.dart`
  entrypoint, which in turn requires `widgetbook` as a regular dependency
  (unreachable from `lib/main.dart`, so tree-shaken from the release binary).
  Canary is app-local `CanaryButton`, not a placeholder `AppButton` — the real
  `AppButton` and other `app_ui` components are registered in E17.
