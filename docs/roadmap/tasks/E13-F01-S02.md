## Notes

- Completed: 2026-04-27
- Result: T-2026-04-27-036 — three commits on `feat/app-button-icon`: `0e64fe3` (AppButton refactor), `615cfd6` (AppIconButton), `104ca10` (login.vue migration). UI tests 272/272; web tests 17/17; lint + typecheck clean.
- Prop API: dropped the `color` axis. `variant: 'primary' | 'secondary' | 'ghost' | 'destructive'` is the single semantic axis matching the bundle's `.btn-*` classes. `iconLeading` and `iconTrailing` are now typed as `IconName` literals (the union from E13-F01-S01) — free strings rejected at compile time.
- Loading state surfaces as `data-loading="true"` + `aria-busy="true"`; the spinner is painted via `::after` (matches the bundle CSS contract: `.btn[data-loading="true"]::after`). Old `app-button--loading` class removed.
- Token-name deviations (the brief named tokens that didn't actually exist in `apps/web/app/assets/css/tokens.generated.css`):
  - `--surface-surface-alt` → `--surface-raised`
  - `--status-danger` → `--status-error-fg`
  - `--text-fg-inverse` → `--text-inverse`
  All three substitutions are documented inline in the SCSS.
- `AppIcon` kept (used by `AppChip`, `AppBadge`); a follow-up cleanup story can migrate those to `IconCS` later.
- A11y: `aria-busy` while loading, `aria-disabled` on disabled, `:focus-visible` outline. AppIconButton requires `aria-label` (TypeScript-enforced) since the icon has no text.
