- Middleware decision matrix: (1) `hasUsers === false` → force `/setup`; (2) `hasUsers === true` and target is `/setup` → redirect to `/sign-in` (setup is locked); (3) public routes (`/sign-in`, `/signup`, `/setup`, `/__tokens`) pass through; (4) authenticated → continue, otherwise `/sign-in`. The `hasUsers` probe is cached per-session via `app/composables/useHasUsersCache.ts` (the cache ref + `resetHasUsersCache()` helper live there so middleware keeps its single-default-export contract).
- Real `cs-web-auth` design lands in E14-F02-S01 (Stage B). This card is the minimal-but-functional auth shell.
