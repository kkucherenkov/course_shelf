## Notes

- Drift is the uncontested local-DB choice (no sqflite/isar/hive anywhere). Adds `drift` + `drift_dev` deps; `build_runner` already present. Consumers are BLoCs (get_it/flutter_bloc), not widgets.
- Completed: 2026-07-16
- Result: (PR pending)
- The three outboxes are deliberately asymmetric, which the card's "DAOs per
  table" wording hides: `progress_outbox` and `notes_outbox` coalesce on
  `lessonId` (their endpoints are idempotent and `/progress/batch` caps at
  `maxItems: 200`, which an append-only table would breach after ~33 min offline
  on one lesson); `bookmarks_outbox` is append-only because the server assigns
  the bookmark id, so offline create-then-edit is resolved by op collapsing on a
  client `localId` (E20 performs the collapse; this story stores the columns).
- Five DAOs, not seven: the three cache tables are one read aggregate
  (`GET /courses/{id}/outline` returns them together).
- `cached_*` store the generated DTO as a JSON `payload` plus promoted query
  columns, so `openapi.yaml` changes flow through codegen without a migration.
- `downloaded_lessons` carries the per-file AES-GCM `nonce` but NOT the key —
  E19 keeps a device-bound key in secure storage.
- CI codegen order is load-bearing: `build_runner` must run before
  `dart run slang`; `slang_build_runner` skips generation but build_runner
  errors on an existing `strings.g.dart`.
