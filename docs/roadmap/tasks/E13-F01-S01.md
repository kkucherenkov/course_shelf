- The component CSS class is `.app-icon-cs` (not `.icon-cs`) to satisfy the project Stylelint `selector-class-pattern` rule requiring an `app-` prefix in `packages/ui` components.
- ESLint filename-case override added for `packages/ui/src/components/IconCS/**` ‚Äî `IconCS` carries an all-caps acronym (`CS`) that neither kebab-case nor PascalCase-with-uppercase-tail accepts cleanly. Scoped override only.
