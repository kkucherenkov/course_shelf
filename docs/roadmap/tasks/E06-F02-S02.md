## Notes

- Completed: 2026-04-26
- Result: T-2026-04-26-026 — commit `4368123` on `feat/ffprobe-thumbnails`. Backend tests 631/631; lint + typecheck clean.
- Deviation from card: skipped `fluent-ffmpeg` in favour of a thin `child_process.execFile` wrapper. Reason: avoids extra dep + types fight; explicit args (no shell) keep us safer against path-traversal-style command injection.
- Thumbnail naming `<videoSourceWithoutExt>.thumb.jpg` next to the source; `*.thumb.jpg` is now classified as `ignored` in stem-match so generated thumbs do not round-trip into `Lesson.materials`.
- Probe + thumbnail each have a 30s `execFile` timeout. Failure on a single file records a `ScanError` (`ffmpeg-probe-failed` / `ffmpeg-thumbnail-failed`) and the scan continues.
- `Lesson` row insertion remains deferred to a future "scan-materialise" story — this card only populates `Scan.discoveredLessons[].metadata` and writes the JPEG.
