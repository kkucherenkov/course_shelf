## Goal

NestJS DTO classes with `class-validator` decorators inferred from schema constraints, exported via the `@app/api-client-ts/server` subpath export.

## Acceptance

- `pnpm spec:codegen` writes classes under `packages/api-client-ts/src/generated/server/`
- Each class extends nothing and uses only `class-validator` and `class-transformer` decorators
- `import { ... } from '@app/api-client-ts/server'` works from `apps/backend`
- A unit test instantiates a generated DTO from a fixture and `validateSync` returns expected violations
