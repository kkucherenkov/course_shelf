## Notes

- Completed: 2026-04-27
- Result: T-2026-04-27-041 — part of primitives batch B. UI tests +26 (13 dialog + 13 command palette).
- AppDialog uses native `<dialog>.showModal()` so the focus trap and `inert` semantics on the rest of the document are browser-native — no userland focus-trap needed. JSDOM doesn't fully implement `<dialog>` so the spec stubs `HTMLElement.prototype.showModal/close` with no-op functions that toggle `open` and dispatch the `close` event; a `stubDialogElement(el)` utility is provided for tests that want to spy on calls.
- AppDialog backdrop click is detected via `event.target === dialogRef.value` (clicking the backdrop hits the dialog itself; clicking inside hits descendants). Only fires `close` when `dismissible: true` (default).
- AppCommandPalette renders inside an AppDialog (size sm), provides a search-and-filter list with arrow-key nav, Enter to select, ESC to close. `Command` type exported alongside as `import type { Command } from '@app/ui'`. Optional `Ctrl/Cmd+K` global shortcut wiring is left to callers.
