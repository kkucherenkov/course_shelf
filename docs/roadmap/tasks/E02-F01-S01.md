## Notes

- Completed: 2026-04-27 (retroactive flip via T-2026-04-27-030)
- Deviation 1: validation uses **Redocly CLI** (`@redocly/cli` 2.x) with `redocly.yaml` rather than Spectral with `.spectral.yaml`. Same fail-on-error behaviour against the same OpenAPI 3.1 doc; AsyncAPI is validated separately via `@asyncapi/cli`. Choosing Redocly avoided a second linter dependency since Redocly already powers `spec:bundle` (`@redocly/openapi-core`). Net result: one toolchain instead of two.
- Deviation 2: `openapi.yaml` is a single inline document, not a tree of `paths/*` + `components/*` `$ref` files. The bundle step still produces a flat `dist/openapi.json`. Splitting into per-path files was deferred — the inline doc is large but legible, and editor `$ref` resolution between files would be a chore for marginal review-ergonomics gain. Will revisit if the file passes ~5000 lines.
